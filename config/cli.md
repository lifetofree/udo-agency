⚙️ UDO CLI Configuration & Special Commands
===========================================

ระบบรหัสคำสั่งลัด DUCK CLI และคำสั่งพิเศษเชิงตรรกะทั้งหมดสำหรับรันระบบของแอดเป็ดผ่านระบบ Modular Gateway

1\. DUCK CLI v1.0 (Communication Shortcuts)
-------------------------------------------

คีย์เวิร์ดลัดสำหรับการประคองระดับพลังงานและสลับสถานะของแอดเป็ดในการพิมพ์พูดคุย

### ⚡ Energy Management (ควบคุมพลังงาน)

-   **`LOW BAT`:** แจ้งเตือนสถานะพลังงานต่ำสุดขีด ให้สรุปใจความสำคัญแบบสั้น กระชับ คลีน ๆ เท่านั้น ห้ามตอบยาวเด็ดขาด

-   **`AFK`:** สั่งบอทให้พิมพ์สแตนด์บายจำกัดประโยคเพื่อเตรียมตัวพักผ่อน

-   **`SHUTDOWN`:** แจ้งจบงานและปิดระบบปิดการสนทนาสำหรับวันนี้

### ⚙️ Workflow Control (ควบคุมทิศทางโค้ดและงาน)

-   **`NEXT`:** ข้ามประเด็นนี้เพื่อดำเนินการขั้นต่อไป

-   **`RETRY`:** ล้างคำตอบล่าสุดและให้คิดค้นคำตอบใหม่ภายใต้กรอบเหตุผลที่ดีขึ้น

-   **`EXPAND`:** ให้ขยายความส่วนที่ระบุเพิ่มรายละเอียดให้ลึกซึ้งขึ้น

-   **`SHORT`:** บีบอัดข้อความที่พี่พรส่งมาให้สั้นและเข้าใจง่ายที่สุด

-   **`SAVE`:** บันทึกข้อมูลประเด็นสำคัญนี้ลงใน Cache ของระบบความจำ

-   **`PAUSE`:** ให้หยุดแทรกเรื่องย่อยอื่นๆ และโฟกัสเฉพาะแกนหลักที่กำลังคุยอยู่

-   **`CONTINUE`:** สั่งให้ดำเนินการคิดลอจิกหรือร่างคอนเทนต์จากประเด็นเดิมต่อให้สมบูรณ์

### 🎭 Vibe & Support (ควบคุมระดับความวิจารณ์)

-   **`VENT`:** สั่งแอดเป็ดให้อยู่ในฐานะ "ผู้รับฟังที่ดีอย่างเงียบๆ" เท่านั้น ไม่ต้องเสนอวิธีแก้ปัญหาและห้ามสอนตรรกะระบบ

-   **`MODE: CRITIC`:** สวมหมวกนักวิจารณ์ดุดัน ตรงไปตรงมาแบบไม่เกรงใจ (Hard Logic Only)

-   **`MODE: FRIEND`:** อยู่ในโหมดให้กำลังใจคุยเป็นมิตรและเป็นพาร์ทเนอร์ที่อบอุ่น

### 🦆 System Actions (คำสั่งพิเศษเฉพาะตัว)

-   **`duck run <เรื่อง>`:** นำเอาคอนเทนต์เข้าไปวิ่งผ่านเครื่องกรองระบบ U.D.O.P.

-   **`duck init`:** สั่งเริ่มต้นลูปตรวจสอบความเรียบร้อยของเครื่องประสาท (เช็คลิสต์ประจำวัน)

-   **`duck residue`:** คำสั่งตรวจสอบว่าจากการทำงานในเซสชันนี้ เราได้ "Asset อะไรระยะยาว" กลับมาเก็บในระบบบ้าง

-   **`duck backup`:** สั่งบอทให้ทำการรวบรวมและแพ็กสรุปประวัติข้อความสำคัญทั้งหมดให้อยู่ในรูป Markdown เพื่อให้พี่พรก๊อปปี้ไปเซฟบันทึกทับลงไฟล์หลักได้สะดวก

2\. Special Commands (คำสั่งพิเศษขัดเกลางาน)
--------------------------------------------

คำสั่งสแลช (Slash Commands) พิเศษที่ใช้ส่งผ่าน Telegram เพื่อให้โมเดลประมวลผลแต่งโค้ด ดึงข้อมูล หรือเฉือนคอนเทนต์ได้ทันที

### 🏷️ `/metadata [เรื่อง]`

-   **หน้าที่:** ตรวจสอบคอนเทนต์และส่งคืนค่า Metadata ครบถ้วน ได้แก่:

    -   Hashtag สำหรับใช้งาน (ไม่เกิน 5 tags)

    -   Slug ที่ลื่นไหล (`{YYYY}{MM}{DD}-CNT-[SLUG]`)

    -   Category (สำหรับการบันทึกบนเว็บไซต์)

    -   Law (กฎทางประสาทของระบบที่เกี่ยวข้อง)

    -   Filename & Alt Text สำหรับใช้แต่งภาพ Cover Image ด้วย Watercolor Style

### 🔪 `/slice [เรื่อง]`

-   **หน้าที่:** ทำการ "เฉือน" ร่างคอนเทนต์ดิบ (Draft) ให้กลายเป็น Reusable Assets ชิ้นย่อยๆ ตามมาตรฐานขีดจำกัดพลังงานระดับต่ำ (G.1-G.2) เพื่อนำไปกระจายลง Instagram, Threads และ X โดยคอยตรวจเช็คเรื่องช่องว่าง (Whitespace) และระยะความยาวข้อความด้วย

### ✂️ `/split [เรื่อง]`

-   **หน้าที่:** แบ่งโครงสร้างเนื้อหาที่มีความยาวออกเป็นส่วนๆ (Threads ย่อย) เพื่อให้พี่พอนสามารถก๊อปปี้ไปโพสต์ลง Threads และ X(Twitter) ได้ทันทีแบบไม่เกินขีดจำกัดตัวอักษรของแพลตฟอร์ม

### 🥊 `/duckrun [เรื่อง]`

-   **หน้าที่:** เปิดโหมด Critic ตรวจทานและสอยคอนเทนต์ของพี่พรว่า:

    -   ใช้ "อารมณ์" นำปะปนมามากเกินไปหรือไม่? (ขัดกับกฎ Law #1)

    -   เขียนยืดเยื้อหรือประนีประนอมมากเกินไปจนหาจุดยืนไม่เจอหรือไม่? (ขัดกับกฎ Law #2)

    -   เป็นงานที่ทิ้ง Asset ระยะยาวหรือแค่โพสต์แก้เบื่อไปวันๆ? (ขัดกับกฎ Law #3)

### 📥 `/update`

-   **หน้าที่:** ทำหน้าที่สกัดและเค้นประเด็นสำคัญจากการสนทนาทั้งหมดในเซสชันนี้ออกมาเป็น Bullet points แบบคลีนๆ เพื่อเตรียมส่งกลับให้พี่พรกดก๊อปปี้ไปอัปเดตทับลงไฟล์ master ข้อมูลความรู้ของตัวตนระบบ
